# TASK-075~081 — 서술형 AI 요약·근거자료 연결 프로그램

_작성일: 2026-07-11 · 상태: TASK-075 활성_

## 1. 사용자 승인과 목표

사용자는 실제 개발 DB의 v4 결과를 확인한 뒤, 현재 AI 작성 영역이 두 필드에
불과하고 일반론적이어서 AI가 이슈를 요약했다는 가치가 충분히 드러나지 않는다고
판단했다. 사용자는 다음 방향을 명시적으로 승인했다.

- AI 요약을 더 자연스럽고 풍부한 서술형으로 확장한다.
- 사용자가 실제 결과를 보고 품질을 최종 판단할 수 있게 한다.
- 검증된 기사, 공식 발표, 공개 문서로 이동하는 링크를 제공한다.
- 개발 DB에서 새 결과를 생성하고 실제 화면으로 검토한다.

이 승인은 기존의 두 필드 작성 제한과 공개 API/UI 계약을 TASK-075~081 범위에서
변경한다. 기존 금지 표현, 미래 결과 단정 금지, 외부 정보와 수치 변화 사이의 원인
단정 금지, 근거 밖 사실 생성 금지, fail-closed, 이전 정상 결과 유지, 검증된 출처만
공개하는 원칙은 변경하지 않는다.

## 2. 제품 계약

### 2.1 AI 작성 영역

새 보고서는 사용자가 원문 질문과 여러 수치를 따로 읽지 않아도 되도록 다음을
자연스러운 한국어로 정리한다.

1. `executive_summary`: 현재 값과 이슈 의미를 설명하는 핵심 요약
2. `current_data_interpretation`: 저장된 값과 24시간/7일 변화의 중립적 해설
3. `conditional_scenarios[]`: 3~4개의 제목 있는 조건부 전개
4. `factors_to_check[]`: 기대값과 함께 확인할 구체적 요인
5. `signals_to_watch[]`: 이후 확인할 공식 자료와 검증 가능한 상태 변화
6. `evidence_synthesis`: 검증된 공개 정보가 있을 때만 작성하는 근거 종합

`relationship_boundary`, `data_limitations`, `caution_note`는 결정적 규칙으로 조립해
항상 표시한다. 숫자·날짜·고유명사·출처 주장은 저장된 근거에서만 가져온다.
상세 문장 구조와 품질 기준은
`reports/task-077-ai-summary-reference-format.md`를 따른다.

### 2.2 자유도와 차단 기준

여기서 서술형은 문장 구성과 설명 순서의 자유도를 뜻하며, 근거 없는 개방형 분석을
뜻하지 않는다. 다음 결과는 저장하지 않는다.

- 다른 이슈에도 그대로 붙일 수 있는 일반론
- 입력에 없는 인물, 날짜, 수치, 사건 또는 출처
- 외부 정보가 관찰된 움직임을 만들었다는 표현
- 현실의 결과를 미리 확정하거나 행동을 유도하는 표현
- 금지 표현, 어색한 미완성 문장, 필드 간 중복
- 수치·근거 ID·출처 URL이 저장 데이터와 일치하지 않는 결과

### 2.3 출처 링크

검증된 후보가 있을 때 자료별로 제목, 기관/매체, 발행 시각, 자료 유형, 중립 요약,
원문 URL을 표시한다. UI 동작은 `원문 확인`, `공식 자료 확인`, `기사 원문 확인`
처럼 자료 유형을 드러내며 새 탭에서 안전하게 연다.

검증된 자료가 없을 때 임의 검색 결과나 미검증 URL로 연결하지 않는다. 대신 이번
검토 구간에 공개 기준을 통과한 자료가 없고 수치 변화의 배경은 확인되지 않았다는
상태를 명시하고 해석 기준 문서로만 연결한다.

## 4. 완료 조건

### TASK-075

- 승인 범위와 유지되는 안전 경계를 ADR에 기록한다.
- Service/Technical/UX Design과 프로젝트 헌장을 갱신한다.
- TASK-075~081의 소유권, 순서, 계약을 문서화한다.

### TASK-076

- v5 구조화 출력과 프롬프트를 구현한다.
- 이슈 고유성, 근거 일치, 중복, 한국어 완결성 검사를 추가한다.
- 공개 정보 0건과 1~3건 입력을 모두 지원한다.
- 실패 시 이전 정상 보고서를 유지한다.

### TASK-077

- 인물·기관·국가·추적 조건 중심 질의를 생성한다.
- 시장 항목 페이지, 전망성 페이지, 검토 구간 밖 자료를 제외한다.
- 공식 단일 출처 또는 독립 복수 출처 검증 규칙은 유지한다.
- 탈락 이유와 출처 확보율을 평가 보고서로 남긴다.

### TASK-078

- v5 응답은 AI 작성 영역과 결정적 안전 영역을 구분한다.
- 모든 주장과 출처가 저장된 근거에 연결되는지 읽기 시점에 재검증한다.
- 구버전/불완전/불일치 보고서는 사용자 화면에 제공하지 않는다.
- 별도 스키마 변경 없이 기존 JSONB를 우선 사용한다.

### TASK-079

- `AI 이슈 요약`을 화면에서 명확히 구분한다.
- 공개 정보 카드에 자료 메타데이터와 안전한 원문 링크를 제공한다.
- 근거 없음 상태를 숨기지 않고 설명한다.
- 데이터 기준 시각과 해석 주의를 같은 화면에 유지한다.

### TASK-080

- 대표 이슈 최소 10건을 개발 DB에 재생성한다.
- 수치·출처·문장·링크 불일치와 금지 표현이 0건이어야 한다.
- 모바일/데스크톱, 근거 0/1/3건, 로딩/실패 상태를 검증한다.
- 외부 호출과 개발 DB 쓰기는 기존 USD 100 프로그램 잔여 범위 안에서만 수행한다.

### TASK-081

- 실제 근거 포함 5건과 근거 없음 5건을 사용자가 직접 열어본다.
- 기존 v4와 v5 결과, 출처, 비용, 실패율을 비교한다.
- 사용자가 유지, 수정, 철회 여부를 최종 판단할 수 있게 한다.

## 5. 승인 제외 범위

이번 승인에는 배포, 운영/프로덕션 DB 쓰기, 인프라 변경, 새 외부 의존성, 기존
migration 수정, 개인 참여자 데이터, 로그인/저장/알림 기능이 포함되지 않는다.
그 작업은 별도 승인이 필요하다.
