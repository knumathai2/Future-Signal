# TASK-077 — AI 이슈 요약 사용자 기준 형식

_작성일: 2026-07-11 · 상태: 사용자 예시 반영_

## 1. 사용자 의도

사용자가 원하는 결과는 짧은 템플릿 카드가 아니라, 한 이슈를 처음 접한 사람이
현재 수치, 조건부 전개, 확인할 자료, 불확실성을 한 번에 이해할 수 있는 구조화된
브리핑이다. 사용자가 제시한 미국–이란 휴전, 기준금리 경로, AI 감독 법안 예시의
공통 구조를 v5 출력의 기준으로 삼는다.

## 2. 공개 순서

1. 이슈 제목
2. `핵심 요약`
3. `현재 데이터 해석`
4. `가능한 조건부 시나리오`
5. `기대값과 함께 확인할 주요 요인`
6. `앞으로 확인할 자료와 변화`
7. `검증된 기사·공식자료`
8. `해석상 주의사항`

## 3. 필드 계약

### `executive_summary`

- 현재 값, 무엇을 다루는 이슈인지, 현재 데이터가 어느 쪽에 상대적으로 더 큰
  비중을 반영하는지 설명한다.
- 수치가 현실 결과를 확정하지 않는다는 경계를 같은 영역에 포함한다.
- 2~4개 문단, 전체 3~6문장을 허용한다.

### `current_data_interpretation`

- 현재 값과 24시간/7일 비교값을 자연어로 해설한다.
- 변화 폭이 큰지, 제한적인지, 데이터가 부족한지를 저장된 임계값으로만 설명한다.
- 참여자들이 특정 이유로 판단했다고 추정하지 않는다.

### `conditional_scenarios`

- 3~4개 시나리오를 제목과 1~3개 문단으로 구성한다.
- 각 시나리오는 `만약 ... 확인된다면`이라는 조건부 설명이어야 한다.
- 현실 결과가 일어날 것이라고 단정하거나 시나리오의 발생 비중을 임의로 배정하지
  않는다.
- 검증된 자료 또는 시장 판정 조건에 없는 세부 사건을 만들어내지 않는다.

### `factors_to_check`

- 기대값과 함께 확인할 공식 발표, 문서, 일정, 지표를 목록으로 제공한다.
- 이 항목들은 변화의 원인이 아니라 후속 확인 대상이다.
- 입력 카테고리와 판정 조건에 구체적으로 연결된 항목만 허용한다.

### `signals_to_watch`

- 실제 행동 또는 검증 가능한 상태 변화에 초점을 둔다.
- 발언만 있는 상태와 공식 일정·문서·행동이 확인된 상태를 구분한다.
- `신호`는 관찰 항목이라는 설명과 함께 사용하며 결과를 미리 확정하지 않는다.

### `evidence_synthesis`

- 같은 검토 구간의 검증된 후보가 있을 때만 작성한다.
- 자료 제목, 발행 기관, 시각, 저장된 중립 요약의 범위를 넘지 않는다.
- 각 서술은 공개 source card와 동일한 candidate/source ID에 연결된다.
- 후보가 없으면 `null`이며 UI는 검증 자료 없음 상태를 별도로 표시한다.

### 결정적 필드

`relationship_boundary`, `data_limitations`, `caution_note`는 코드가 저장 근거로
조립한다. 모델이 바꾸거나 생략할 수 없다.

## 4. 사용자 예시에서 유지할 특성

- 일반 독자가 이해할 수 있는 제목과 섹션 순서
- 현재 값의 의미와 한계를 같은 문맥에서 설명
- 단일 결론 대신 복수의 조건부 전개 제시
- 후속 발표, 일정, 문서, 데이터 지표를 구체적으로 목록화
- 단순 발언과 실제 문서·행동을 구분
- 마지막에 데이터 대표성, 변동성, 현실 결과 비확정 경계 제시

## 5. 안전 조정

다음 표현 방식은 의미를 보존하면서 조정한다.

| 사용자 예시의 의도 | v5 표현 원칙 |
|---|---|
| 기대값을 변화시킬 요인 | 기대값과 함께 확인할 요인 |
| 가능성을 높이거나 낮추는 신호 | 조건부로 함께 관찰할 자료 또는 상태 변화 |
| 시장이 특정 사건에 반응할 것 | 해당 자료 공개 전후의 값 변화를 별도로 확인 |
| 기본 시나리오 | 현재 값에 상대적으로 더 큰 비중이 반영된 조건부 경로 |
| 미래 결과 설명 | 판정 조건이 충족·부분 충족·미충족될 경우의 의미 설명 |

검증된 근거 없이 군사 행동, 정책 결정, 법안 조항, 경제지표의 향후 방향을 새로
만들지 않는다. 행동을 유도하거나 특정 결과를 지지하는 표현도 생성하지 않는다.

## 6. 품질 판정

다음 중 하나라도 해당하면 사용자 검토 대상으로 제공하기 전에 실패 처리한다.

- 제목과 고유명사를 바꾸면 다른 이슈에도 그대로 적용되는 문장
- 수치만 반복하고 해석 또는 확인 항목이 없는 결과
- 시나리오 간 실질적 차이가 없는 결과
- 입력 근거에 없는 구체적 사건, 일정, 조항, 지표 방향
- 검증 후보가 없는데 기사나 외부 사건을 근거처럼 서술한 결과
- 조건부 표현 없이 미래 상태를 단정한 결과
- 섹션 간 문장 중복 또는 어색한 미완성 한국어

## 7. 구현 반영

TASK-076의 초기 여섯 필드 구현은 이 사용자 기준에 맞춰 다음 계약으로 조정한다.

```text
executive_summary
current_data_interpretation
conditional_scenarios[]
factors_to_check[]
signals_to_watch[]
evidence_synthesis|null
+ deterministic relationship_boundary/data_limitations/caution_note
```

TASK-077은 각 목록과 시나리오가 실제 시장 메타데이터 및 검증된 공개자료에서
구체성을 얻도록 입력 품질을 개선한다. TASK-078 API와 TASK-079 UI는 위 순서를
그대로 공개한다.
