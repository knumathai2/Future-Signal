있고, 삭제된 상세 자료는 Git 이력에서 확인할 수 있습니다.

## 현재 v8 전환 상태

v8은 v7의 근거, 출처-상위항목 연결, 요청, 임대, 캐시, 이전 정상 결과 유지, 기준
시각, 한계, 주의 문구와 공개 차단 규칙을 유지합니다. 작성 구조는 이슈 중심 제목,
요약, 유형이 지정된 2~6개 섹션으로 바뀌었습니다. TASK-113~117은 제한된 범위의
출처 탐색 확대, 안전한 재시도, 문맥 기반 문구 검증, 개별 검증 NDJSON 블록과 SSE
전달을 추가했습니다.

현재 API는 성공한 v8 행만 복원합니다. 명시적 승인에 따라 현재 로컬·개발
데이터베이스에서는 저장된 v1~v7 보고서와 v7 요청·이벤트 이력을 제거했습니다.
v8 보고서·요청·이벤트·블록과 지표·스냅샷·맥락·정의 근거는 유지했습니다. 운영
데이터베이스는 이 정리 작업으로 변경하지 않았습니다.

## 계속 적용되는 원칙

- 공개 집계 데이터만 사용합니다.
- 미래 결과나 외부 자료와 관측 변화의 관계를 근거 없이 단정하지 않습니다.
- 모든 데이터 화면에 기준 시각과 해석 주의사항을 표시합니다.
- 출처 URL, 제목, 근거 문장, 지표, 날짜, 대상, 근거 참조와 입력 지문을 정확히
  복원할 수 있어야 합니다.
- 안전하지 않은 링크나 일치하지 않는 근거는 공개를 차단합니다.
- 제공자 또는 검증 실패 시 이전의 정상 결과를 유지합니다.
- 현재 문구 정책과 출처 표시 정책을 적용합니다.
- 배포, 운영 데이터 쓰기, 의존성, 스키마, 공개 API, 인프라, 문구 정책 변경에는
  기존 승인 절차를 적용합니다.
